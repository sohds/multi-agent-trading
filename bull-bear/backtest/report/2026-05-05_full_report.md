## 0. TL;DR

오늘 진행한 Bull-Bear 백테스팅 실험을 통해 다음을 확인:

1. **인프라 검증 통과** — pykrx as_of, temperature=0, GT 분포 모두 정상
2. **시스템 프롬프트 비대칭이 존재함** — Bear가 "매도/관망" 둘 다 허용돼 사실상 wait-and-see 에이전트로 작동
3. **Bear 편향 71%가 종목 무관하게 일관됨** — 데이터 차이가 아닌 프롬프트 문제 확정
4. **프롬프트 수정(옵션 C) 후 균형 회복** — bearish 71% → 49%, bullish 11% → 31%
5. **하지만 적중률은 오히려 하락** — 기존 결과는 "Bear의 default wait-and-see가 우연히 맞은 것"이었음을 시사
6. **마스킹은 정보 손실의 주범 아님** (Phase 2-nomask 검증) — 마스킹 OFF로도 적중률 +1~3%p 미미한 개선
7. **본질적 한계는 Technical 지표 자체** — 30~42% 적중률은 정보량 부족이 원인. Macro/Sector 추가가 다음 검증 단계

## 5. 프롬프트 분석 — 비대칭 발견

`bear_agent.py`와 `bull_agent.py` 시스템 프롬프트 비교 시 **3가지 비대칭** 식별:

### 비대칭 ① — Summary 문구

```python
# bull_agent.py line 34
"summary": "<매수 관점 한 줄 요약>"

# bear_agent.py line 34
"summary": "<매도/관망 관점 한 줄 요약>"   # ← "관망" 허용
```

> Bull은 "매수"만 허용, Bear는 "매도 OR 관망" 둘 다 허용. **결정적 단서**.

### 비대칭 ② — 역할 정의에 "불확실성"

```
# bear_agent.py line 14
"리스크 요인, 고평가 구간, 수급 이탈, 악화 추세, 불확실성을 부각합니다."
                                              ^^^^^^
                                              항상 적용 가능 (default fallback)
```

"불확실성"은 너무 광범위해 매도 신호가 약할 때도 default로 사용 가능.

### 비대칭 ③ — Confidence 의미가 모호

기존 프롬프트엔 confidence 산출 룰이 없어, 모델이 자유롭게 "내 자신감"으로 해석함. Bear가 "관망" 입장으로 도피하면 confidence를 낮출 이유가 없어 일관되게 0.8+ 출력됨.

## 6. 프롬프트 수정 — 옵션 C 채택

세 가지 설계 대안 검토 후 **옵션 C** 선택:

| 옵션 | 설계 |
|------|------|
| A | Bull=매수, Bear=매도, 관망은 시스템 레벨 |
| B | Bull/Bear 양쪽 관망 허용 (escape hatch 우려) |
| **C** | **Bull/Bear stance 유지 + Confidence를 "신호 강도"로 재정의** |

옵션 C 핵심:
- Bull/Bear stance 고정 (각자 maximally adversarial)
- Confidence = "내가 자신 있다" → "**신호가 얼마나 강한가**"
- 양쪽 confidence 모두 낮으면 → 시스템(`backtest_runner` neutral 분류)이 자동 "관망" 판정
- "관망" 키워드는 Bull/Bear 양쪽 모두 사용 금지

### 적용된 변경

1. **Bear summary**: "매도/관망" → "매도"만
2. **Bear 역할**: "불확실성" 제거 → 구체적 매도 신호 (고평가, 수급 이탈, 추세 약화, 모멘텀 둔화, 과매수)로 교체
3. **Confidence 산출 룰** (Bull/Bear 양쪽):
   ```
   - 0.8+: 신호 3개 이상이 같은 방향 강하게 가리킬 때
   - 0.6~0.8: 핵심 신호 1~2개가 명확할 때
   - 0.5~0.6: 신호가 혼재되거나 약할 때
   - 0.5 미만: 근거가 부족할 때 (필수)
   ```
4. **명시적 지시**: "관망", "두고 보자", "신중하게 접근" 같은 wait-and-see 입장 사용 금지. 근거 약하면 confidence를 0.3~0.5로 낮추고 솔직히 표현.

## 9. 핵심 발견 — 무엇이 진짜 의미인가

### 발견 1: 시스템 프롬프트 비대칭은 명백한 버그였다

수정 전 결과의 71% Bear 편향은 **모델 성능이 아닌 프롬프트 문구의 결과**였다. 프롬프트 수정만으로 22%p 변동.

### 발견 2: 수정 전 81% Bear 정확도는 환상이었다

Bear가 "매도/관망 OR" 둘 다 허용된 상태에서 **사실상 wait-and-see 에이전트로 작동**. wait-and-see는 어떤 시장에서도 어느 정도 합리적이라 우연히 적중. 진짜 매도 신호 식별 능력이 50%(랜덤)이라는 것이 수정 후 드러남.

### 발견 3: 한계의 원인은 마스킹이 아닌 Technical 지표 자체

Phase 2-nomask로 마스킹의 영향을 ablation한 결과, 마스킹 OFF로도 N=5 +1.3%p, N=20 +2.9%p의 미미한 개선만 확인됐다. **마스킹은 정보 손실의 주범이 아니다.**

수정 후 적중률 N=10/20에서 큰 폭 하락한 것은 **에이전트가 진짜 신호를 따라가려 하지만 Technical 지표만으로는 정보가 부족**해서 발생. 다음 검증 후보:
- Macro/Sector 컨텍스트 추가 (트랙 C) → 가장 유력
- ROUNDS=2 토론 → 신호 재검증 효과
- 다른 모델 시도 (gpt-5, claude-sonnet 등)

### 발견 4: KB금융이 가장 큰 개선 (+18.7%p)

KB금융은 Phase 0에서 N=20 GT가 bullish 17/4 (81% 상승)으로 **명확한 강세장**이었음. 수정 전엔 Bear 13개 예측으로 매수 못 잡음. 수정 후 Bull 6→9로 늘면서 강세 신호 일부 포착. **명확한 추세에서는 수정된 프롬프트가 더 잘 작동함**을 시사.

### 발견 5: Confidence 룰이 작동함

Bull/Bear 평균 confidence가 모두 하락 (Bear -0.21, Bull -0.14). **모델이 신호 약할 때 솔직히 confidence 낮춤**. 특히 Bear가 더 크게 하락한 것은 wait-and-see 도피처 제거의 직접 효과.
