## 1. 실험 개요

### 목적

Bull/Bear 에이전트가 실제 주가 방향성을 예측하는 데 유효한지 검증.  
단계적으로 데이터를 추가하며 어떤 요소가 적중률에 기여하는지 확인.

### 공통 환경

| 항목 | 값 |
|------|-----|
| 모델 | gpt-5.4-mini, temperature=0 |
| 종목 | 삼성전자·현대차·KB금융·삼성바이오로직스 (4종목) |
| 기간 | 2024-01 ~ 2025-12, 매월 1영업일 (24개월) |
| GT 라벨 | N=5/10/20 거래일 기준 bullish/bearish/neutral |
| 총 케이스 | 96개 (4종목 × 24개월) |
| 평가 지표 | 적중률, Balanced Accuracy (Bull/Bear 정확도 평균) |

## 2. 실험 흐름 및 결과

### Phase 0 — 사전 검증 ✅

- `pykrx` 과거 데이터 조회 정상 (룩-어헤드 차단 확인)
- `temperature=0` 지원 확인 (재현성 확보)
- GT 라벨 분포: N=10 기준 bullish 50% / bearish 38% / neutral 12% — **클래스 균형**

---

### Phase 1 / Phase 2 — Technical 지표만, 원본 프롬프트

| 실험 | 케이스 | N=5 | N=10 | N=20 | Bear 비율 |
|------|--------|-----|------|------|-----------|
| Phase 1 (삼성전자) | 24 | 47.1% | 42.1% | 50.0% | 71% |
| Phase 2 (4종목) | 96 | 40.3% | 42.2% | 43.5% | 71% |

**핵심 발견**: Bear가 종목과 무관하게 71%로 일관됨 → 데이터 차이가 아닌 **프롬프트 문제** 확정.

원인 분석:
- Bear summary에 "매도/관망" 둘 다 허용 → 사실상 wait-and-see 에이전트로 작동
- "불확실성"이 default fallback으로 기능 → 매도 신호 없어도 confidence 높게 유지
- Confidence 산출 룰 없음 → Bear가 자유롭게 0.8+ 출력

---

### Phase 2-prompt — 프롬프트 수정 (옵션 C) 적용

**변경 내용**: Bull/Bear stance 유지, Confidence를 "신호 강도"로 재정의, "관망" 표현 금지

| 실험 | N=5 | N=10 | N=20 | Bear 비율 | Bull 비율 |
|------|-----|------|------|-----------|-----------|
| 수정 전 | 40.3% | 42.2% | 43.5% | 71% | 11% |
| **수정 후** | **41.3%** | **33.8%** | **30.4%** | **49%** | **31%** |

- 예측 분포 균형 회복 (Bear 71% → 49%, Bull 11% → 31%)
- **단기(N=5) 유지, 중장기(N=10/20) 하락** → 기존 Bear 정확도 81%는 wait-and-see가 우연히 맞은 것이었음을 시사
- KB금융 N=5 기준 +18.7%p 개선 (강세장 매수 신호 포착 시작)

---

### Phase 2-nomask — 마스킹 OFF 효과 검증

- 마스킹 ON → OFF 시 N=5 +1.3%p, N=20 +2.9%p → **마스킹은 정보 손실의 주범 아님**
- 본질적 한계는 **Technical 지표 자체의 정보량 부족**으로 결론

## 4. 핵심 인사이트

### ① Macro는 유효하다

Phase 2-prompt 대비 N=10 +22.9%p는 단순 노이즈로 볼 수 없는 수준. ECOS 기반 금리·환율·KOSPI 정보가 단기 방향성 판단에 실질적으로 기여함.

### ② Sector 신호는 시간 지평 미스매치 문제가 있다

PER/PBR 같은 valuation 신호는 이미 시장에 반영된 공개 정보로, N=10(2주) 방향성과 직접 연결되기 어렵다. 가치 팩터는 수개월~수년 단위로 작동하는 것이 학술적으로 검증된 특성이다. EPS, 수급, 상대강도도 단기보다 중기 신호에 가깝다.

### ③ 더 근본적인 설계 질문이 남아있다

> 이 Bull-Bear 시스템은 어떤 시간 단위의 판단을 지원하려고 만든 것인가?

- **단기 트레이딩(N=10)** 목적이라면: Sector 펀더멘털은 시스템 구조적으로 맞지 않음
- **중기 투자 판단(N=60~120)** 목적이라면: 백테스트 horizon 자체가 잘못 설정된 것

Sector를 백테스트에서만 빼고 실제 시스템에서 쓴다면, 검증한 시스템과 실제 시스템이 달라진다. 이 misalignment를 해소하려면 **팀 차원의 시스템 목적 합의**가 선행되어야 한다.
