## 1. 요약

Phase 3b에서 sector를 추가한 뒤 성능이 죽은 핵심 원인은 sector collector 실패가 아니다. 패치 적용 후 full run 기준 `sector_error`는 0건이었다.

문제는 **sector 신호가 추가되면서 Bull/Bear 판단이 기존 technical/macro 판단에서 벗어났고, 그 벗어난 방향이 N=10 라벨과 맞지 않은 케이스가 많았다**는 점이다.

N=10 기준:

| 구분 | 케이스 수 |
|------|-----------|
| Phase 3a 정답 -> Phase 3b 오답 | 24 |
| Phase 3a 오답 -> Phase 3b 정답 | 15 |
| 둘 다 정답 | 12 |
| 둘 다 오답 | 45 |

즉 sector 추가는 일부 케이스를 살렸지만, 더 많은 케이스를 깨뜨렸다.

## 7. 결론

Sector 추가가 성능을 떨어뜨린 이유는 다음으로 보는 것이 가장 타당하다.

1. Sector payload는 정상 수집됐다.
2. 하지만 sector feature 다수가 N=10 단기 방향성보다 중장기 설명 변수에 가깝다.
3. Bull/Bear 프롬프트가 `저평가`, `EPS 개선`, `외국인/기관 수급`, `상대강도`를 단기 방향 신호로 직접 해석했다.
4. 그 결과 technical/macro가 잡던 단기 과열, 저항선, 추세 이탈 신호가 희석됐다.

따라서 다음 실험은 P3 확장보다 먼저 **sector prompt ablation**이 낫다.

권장:

1. Sector를 넣되 `valuation/EPS`는 단기 방향 신호가 아니라 배경 정보로 제한
2. `수급`은 가격 위치가 저항/과열인지 여부와 함께 해석하도록 프롬프트 수정
3. `상대강도`는 momentum continuation과 mean reversion 가능성을 모두 보게 만들기
4. 동일 Phase 3b를 `sector=relative_strength only`, `sector=supply only`, `sector=valuation only`로 쪼개서 ablation

## 8. 추가 의견 — ablation 실험 전 프롬프트 수정 우선 검토

오답 케이스 리뷰가 이미 충분히 원인을 좁혔기 때문에, 섹션 4의 신호 키워드 분포와 섹션 6의 패턴 분석만 봐도 주범은 `valuation/EPS` 계열 신호임이 분명하다. `sector=valuation only` ablation을 돌려봐야 얻는 정보가 "valuation이 주범이다"는 확인인데, 이건 이미 오답 목록에서 드러난다.

따라서 ablation 실험 4종(~$0.90, ~7.5h)을 먼저 돌리기보다 다음 순서가 더 효율적이다.

**권장 순서**

1. **프롬프트 수정 먼저**: Bull/Bear 프롬프트에 sector 신호의 해석 제약을 명시한다.
   - `valuation(PER/PBR/EPS)`: "단기 방향 신호로 쓰지 말 것. 현재 가격 위치의 배경 맥락으로만 활용."
   - `수급`: "가격이 저항선 근처이거나 과열 구간이면 수급 개선도 단기 차익실현 신호일 수 있음."
   - `상대강도`: "모멘텀 지속 가능성과 평균회귀 가능성을 함께 언급할 것."

2. **수정된 프롬프트로 Phase 3b 재실행** (~$0.30, ~2.5h): Phase 3a 수준 이상으로 회복되면 sector ablation 생략 가능.

3. **그래도 회복 안 되면 그때 ablation**: `RS only` / `supply only` / `valuation only` 순으로 쪼개서 진단.

이 순서라면 프롬프트 수정이 효과 있을 경우 ablation을 통째로 건너뛸 수 있어 비용과 시간 모두 아낄 수 있다.

## 9. 더 근본적인 질문 — 섹터 신호 자체가 N=10에 적합한가

섹션 8의 프롬프트 수정 방향을 논의하는 중, 더 근본적인 문제가 제기됐다.

**핵심 지적**: sector의 valuation 신호(PER/PBR/EPS)는 이미 시장에 공개된 정보로 즉시 가격에 반영되는 경향이 있다. "저평가이므로 N=10 내에 오른다"는 논리가 성립하려면, 시장이 그 정보를 아직 반영하지 못했다는 전제가 필요한데, 이는 단기 예측에서 매우 강한 가정이다.

신호별 N=10 적합성을 정리하면 다음과 같다.

| 신호 | N=10 적합성 | 이유 |
|------|-------------|------|
| valuation (PER/PBR) | 낮음 | 가치 팩터는 수개월~수년 단위로 작동. N=10에서는 이미 가격에 반영 |
| EPS 개선 | 조건부 | 실적 발표 직후라면 drift 효과 존재. 단, 발표 타이밍 정보 없이 사용 시 의미 약함 |
| 수급 (외국인·기관) | 약하게 유효 | 단기 모멘텀 효과 있으나 T+1 데이터라 일부 이미 반영. 노이즈 큼 |
| 상대강도 (RS) | 기간 의존적 | 3~12개월 RS는 모멘텀 팩터로 검증됨. N=10은 모멘텀/평균회귀 경계 구간 |

**결론**: 이는 sector 에이전트가 잘못 설계됐다는 의미가 아니다. sector 분석 자체는 중장기 판단에 유효하다. 문제는 **백테스트 평가 기준(N=10)과 sector 신호의 시간 지평이 맞지 않는다**는 것이다.

이 관점에서 다음 세 가지 방향을 검토할 수 있다.

| 옵션 | 내용 |
|------|------|
| A | 백테스트에서 sector 제외, Phase 3a(Macro만) 상태 유지. sector는 실제 시스템에서는 쓰되 N=10 정확도 평가에서는 제외 |
| B | 평가 horizon을 N=20 이상으로 확장해 sector 신호 재평가 |
| C | sector payload에서 valuation 제거, 수급·RS만 유지 + 프롬프트 제약 |

Phase 3a가 이미 N=10 기준 56.7%로 가장 좋은 결과를 냈다는 점, 그리고 프롬프트 수정으로 valuation 신호의 오남용을 완전히 차단하기 어렵다는 점을 감안하면, **옵션 A가 현 시점에서 가장 현실적인 선택**으로 보인다. sector 에이전트는 시스템 내에서 중장기 판단 근거를 보강하는 역할로 두고, 단기 백테스트 정확도 향상은 Macro + Sentiment 방향으로 이어가는 것이 낫다.
