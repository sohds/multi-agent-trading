# 시스템 프롬프트 개선 제안서

**작성일**: 2026-05-05
**상태**: 옵션 C 채택 (Bull/Bear 입장 유지 + Confidence 의미 명확화)
**근거 실험**: Phase 1 (24 케이스) + Phase 2 (96 케이스) — 트랙 A
**핵심 발견**: Bear 에이전트가 4종목·24개월 일관되게 71% Bear 편향, 적중률 40~50% (랜덤 이하)

---

## 0. 설계 결정 — 옵션 C 채택

세 가지 설계 대안을 검토한 결과 **옵션 C**로 결정:

| 옵션 | 설계 | 채택 |
|------|------|------|
| A | Bull=매수, Bear=매도, 관망은 시스템 레벨 | ❌ |
| B | Bull/Bear 양쪽 관망 허용 | ❌ — 관망이 default escape hatch가 되어 토론 약화 |
| **C** | **Bull/Bear stance 유지 + Confidence를 "신호 강도"로 재정의** | **✅** |

**옵션 C 핵심**:
- Bull stance는 "bullish", Bear stance는 "bearish"로 고정
- Confidence는 "내 자신감"이 아닌 **"내 stance를 뒷받침하는 신호의 강도"**
- 양쪽 confidence가 모두 낮으면 → 시스템(backtest_runner의 neutral 분류 또는 오케스트레이터)이 "관망"으로 판정

이 방식의 장점:
1. Adversarial debate의 본질 유지 (각자 입장 최대 논거)
2. 현재 backtest_runner의 neutral 분류 로직과 일관됨 (`|conf_diff| < 0.05`)
3. 에이전트가 신호 약할 때 솔직히 confidence 낮춤 가능 → "관망" 자연 발생

## 1. 결정적 단서 — 프롬프트 비대칭

`bull_agent.py`와 `bear_agent.py`의 시스템 프롬프트 비교 시 **명백한 비대칭**이 발견됐습니다.

### Bull 프롬프트 (line 34)
```
"summary": "<매수 관점 한 줄 요약>"
```

### Bear 프롬프트 (line 34)
```
"summary": "<매도/관망 관점 한 줄 요약>"
```

> **Bull은 "매수"만 허용, Bear는 "매도 OR 관망" 둘 다 허용.**

이로 인해 Bear는 매도 신호가 약할 때 "관망" 입장으로 도피할 수 있다. 관망은 어떤 시장 상황에서도 합리적으로 들리므로 confidence가 일관되게 높게 산출된다.

또한 Bear의 핵심 역할 정의에 **"불확실성"**(line 14)이 포함돼 있어, 매도 신호가 약해도 "불확실성" 카드로 항상 비관적 입장 유지 가능.

```
Bear: "리스크 요인, 고평가 구간, 수급 이탈, 악화 추세, 불확실성을 부각"
                                                  ^^^^^^
                                                  always applicable
```

## 2. 정량 증거 — Phase 1·2 결과

### Bear summary 패턴 (Phase 1 24케이스)
거의 모든 케이스가 다음 형태:
> "지금은 추세·수급이 강하지 않아 매수보다 관망이 더 합리적입니다."

→ 24개 중 22개에서 "관망" 키워드 등장. **본질적으로 매도 추천이 아닌 wait-and-see**.

### confidence 분포

| 케이스 | Bull conf | Bear conf | 차이 |
|--------|-----------|-----------|------|
| 평균 | ~0.78 | ~0.86 | -0.08 |
| 최대 차이 | — | — | -0.26 |
| Bull 우세 케이스 | — | — | 단 11/96 |

Bull confidence는 0.63~0.90 분포, Bear는 0.68~0.94. **Bear가 평균 0.08 높음** — 이는 데이터 차이가 아니라 프롬프트 비대칭의 결과.

## 3. 개선안 — 3가지 변경

### 변경 ①: Bear summary 정의 수정

**현재**:
```python
"summary": "<매도/관망 관점 한 줄 요약>"
```

**변경 후**:
```python
"summary": "<매도 관점 한 줄 요약>"
```

> "관망" 옵션 제거. Bear는 매도 추천 역할에 집중.

### 변경 ②: Bear 역할 정의에서 "불확실성" 제거

**현재** (line 14):
```
- 리스크 요인, 고평가 구간, 수급 이탈, 악화 추세, 불확실성을 부각합니다.
```

**변경 후**:
```
- 고평가 구간, 수급 이탈, 추세 약화, 모멘텀 둔화 같은 매도 신호를 적극 부각합니다.
```

> "불확실성"은 너무 광범위해 default fallback이 됨. 구체적·검증 가능한 매도 신호로 교체.

### 변경 ③: Confidence 산출 룰 명시

**Bull/Bear 양쪽 시스템 프롬프트에 추가**:
```
[Confidence 산출 룰]
- 0.8 이상: 데이터 3개 이상이 같은 방향 강하게 가리킬 때
- 0.6~0.8: 핵심 데이터 1~2개가 명확하게 가리킬 때
- 0.5~0.6: 신호가 혼재되거나 약할 때
- 0.5 미만: 자신의 stance 근거가 부족할 때 (강제 표현)
```

> 현재는 confidence가 "내가 얼마나 자신감 있게 말하는가"인데, 이를 "신호의 강도"로 재정의.

### (선택) 변경 ④: Bear에게 명시적 지시 추가

```
[중요]
당신은 "매도 신호 식별자"입니다. "관망"이나 "두고 보자"는 입장은 매도 신호가 아니므로 사용하지 마십시오.
매도 신호가 약하다면 confidence를 0.5 이하로 낮추고, 강한 매도 신호가 없다고 솔직히 표현하십시오.
```

수정 후 동일한 96 케이스(Phase 2)를 재실행하여 비교:

| 지표 | Phase 2 (수정 전) | Phase 2-prompt (수정 후) | 기대 변화 |

| Bear 예측 비율 | 71% | **40~55%** | 큰 폭 감소 |
| Bull 예측 비율 | 11% | **30~45%** | 증가 |
| Neutral 비율 | 18% | **10~20%** | 유지 또는 약간 증가 |
| 적중률 (N=10) | 42% | **>55%** 목표 | 상승 |

비용: ~$0.30, 시간 ~1.5시간 (Phase 2와 동일 규모).

## 5. 팀 논의 필요 사항

1. **Bear의 "관망" 입장 허용 여부** — 토론 시스템에서 Bear가 매도 추천만 해야 하는지, 관망도 valid 입장인지 도메인 차원에서 결정 필요
2. **Confidence semantic 변경** — "확신도"에서 "신호 강도"로 의미 변경 시 오케스트레이터 측 영향 (`agreement_score` 산출 시 사용 방식) 검토 필요
3. **수정 후 재실험 시점** — Phase 2-prompt를 즉시 진행할지, 다른 변경(Macro 추가 등)과 묶어서 진행할지

## 7. 위험 요소

- **Bear가 너무 약해질 위험**: 수정 후 Bull 편향이 71% 정도로 뒤집힐 수 있음. Phase 2-prompt 결과로 모니터링 필요.
- **Confidence 룰이 LLM 행동에 미치는 영향 불확실**: 모델이 룰을 따를지 보장 안 됨. 결과 분석에서 confidence 분포 변화도 함께 검증해야.
- **마스킹 영향과의 혼재**: 트랙 A는 마스킹 ON. 마스킹된 가격 정보가 Bull 논거를 약화시켰을 가능성도 잔존. → 필요 시 마스킹 OFF 트랙으로 추가 검증.
